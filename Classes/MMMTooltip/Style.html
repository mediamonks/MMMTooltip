Style

  • Location
    See more
  • location

    Determines the location of the tooltip arrow as well as where the tooltip will appear. For instance, on a topLeading location, the tooltip will appear below the targetView, with the arrow on the top leading side. Whereas on a bottomCenter location, the toolltip will appear above the targetView with the arrow on the bottom center.

  • offset

    The amount we offset the tooltip from the targetView,

  • padding

    The amount of padding that the tooltip should use when positioning itself in the superview, so it doesn’t hug the screen edges.

  • backgroundColor

    The background color of the tooltip and arrow.

  • cornerRadius

    The radius of the tooltip.

  • arrowRadius

    The radius of the arrow. Where 0 would be a sharp rectangle.

  • arrowSize

    The size of the arrow.

  • contentInsets

    How much we should inset the contentView.

  • buttonInsets

    How much we should inset the buttonView.

  • preferredPaddingMultiplier

    How much the arrow and tooltip should try to add as padding. For example on a topLeading location, you usually don’t want the arrow & tooltip all the way on the left. Setting this to 0.3 for instance, will place the arrow at 30% of the width of the tooltip.

  • preferredPaddingPriority

    How hard we should try to stick to the preferredPaddingMultiplier.

  • maxWidthMultiplier

    How wide the tooltip can get according to the superview. This is a multiplier, so at 1.0 the tooltip will take up all the available space in the superview of the targetView, whereas with 0.5 the tooltip will never get bigger than half the size of the superview.

  • init(location:offset:padding:backgroundColor:cornerRadius:arrowRadius:arrowSize:contentInsets:buttonInsets:preferredPaddingMultiplier:preferredPaddingPriority:maxWidthMultiplier:)
